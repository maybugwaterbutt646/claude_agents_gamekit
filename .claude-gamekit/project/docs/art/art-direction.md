# Art Direction

## Intent

Define only the high-level visual direction and placeholder constraints. Detailed asset build instructions belong in the asset catalog or Asset ABI contracts.

## Default Placeholder Style

- Prefer primitives, kitbash, and graybox materials.
- Use placeholders to validate gameplay scale and swap safety, not to chase final quality.
- Keep naming and silhouette stable across a feature slice.

## Hard Rules

- Placeholder naming must stay stable once referenced in docs or artifacts.
- Size and orientation must match the Asset ABI.
- Any extra grouping introduced for readability must be recorded in the asset catalog.
